React with Next.js App Router — Registration, Login, and Recipe CRUD

In this lesson we extend RecipeShare so guests can browse recipes, users can register and log in, and authenticated users can create, edit, and delete their own recipes. We follow proper Next.js conventions: each page lives in its own route, all API calls live in a shared lib/api.js file, and pages stay minimal and easy to read.

Concept: layout, public routes, and protected routes

How layout.js works

In the Next.js App Router, every folder inside app/ can have its own layout.js. The root app/layout.js wraps every single page in the app. It is the perfect place to put things that should always be visible — like a navigation bar or a global footer.

Think of it like a picture frame: the frame (layout) stays the same while the picture (page) changes as the user navigates. Next.js renders the layout once and swaps only the {children} slot when the route changes. 

// app/layout.js — rendered on EVERY page
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <NavBar />       {/* ← always visible */}
        {children}        {/* ← current page goes here */}
      </body>
    </html>
  );
}
Where does NavBar come from? We will build it ourselves in Step 3 as a component in components/NavBar.jsx. It reads auth state from localStorage and shows either Login / Register links for guests or a New Recipe + Logout button for logged-in users. The reason it lives in layout.js — and not in any individual page — is so it appears on every route automatically without having to add it to each page manually.

Public routes vs. protected routes

Not every page needs a logged-in user. We split our routes into two categories:

🌐 Public routes

Anyone can visit these — no login required.

  • / — home page, recipe list
  • /login — login form
  • /register — registration form

These pages just render. No auth check needed.

🔒 Protected routes

Only logged-in users should reach these pages.

  • /recipes/create — create a recipe
  • /recipes/[id]/edit — edit a recipe

Each protected page checks for auth on load. If no auth is found, it immediately calls router.push("/login") — the user never even sees the page content.

User navigates to a route → layout.js wraps it → Public? → render page or Protected? → check auth → redirect to /login if missing
Important: the layout itself does not do the auth check. Each protected page is responsible for guarding itself. This keeps the layout simple and avoids blocking public pages.
Frontend guard ≠ real security. The router.push("/login") guard is purely for user experience. A determined user could bypass it. The real security is on the Flask server — it rejects requests without a valid Authorization header with a 401 Unauthorized response. Always secure the backend, regardless of what the frontend does.

Step 1: create lib/api.js — all API calls in one place

Instead of writing fetch() calls inside every component or page, we put them all in a single file. This makes pages easy to read, and if the backend URL changes we only update one file.

We also put two small helpers here: saveAuth and getAuth to store the logged-in user in localStorage so the session survives a page refresh.

lib/api.js
const BASE_URL = "http://localhost:5000";

// ─── Auth helpers (localStorage) ────────────────────────────────────────────

export function saveAuth(username, password) {
  localStorage.setItem("auth", JSON.stringify({ username, password }));
}

export function getAuth() {
  const raw = localStorage.getItem("auth");
  return raw ? JSON.parse(raw) : null;
}

export function clearAuth() {
  localStorage.removeItem("auth");
}

function authHeader(username, password) {
  return { Authorization: "Basic " + btoa(username + ":" + password) };
}

// ─── Recipes ─────────────────────────────────────────────────────────────────

export async function fetchRecipes() {
  const response = await fetch(`${BASE_URL}/api/recipes`);
  if (!response.ok) throw new Error("Failed to load recipes");
  return response.json();
}

export async function createRecipe(recipeData, username, password) {
  const response = await fetch(`${BASE_URL}/api/recipes`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      ...authHeader(username, password),
    },
    body: JSON.stringify(recipeData),
  });
  if (!response.ok) {
    const data = await response.json();
    throw new Error(data.error || "Could not create recipe");
  }
  return response.json();
}

export async function updateRecipe(id, recipeData, username, password) {
  const response = await fetch(`${BASE_URL}/api/recipes/${id}`, {
    method: "PUT",
    headers: {
      "Content-Type": "application/json",
      ...authHeader(username, password),
    },
    body: JSON.stringify(recipeData),
  });
  if (!response.ok) {
    const data = await response.json();
    throw new Error(data.error || "Could not update recipe");
  }
  return response.json();
}

export async function deleteRecipe(id, username, password) {
  const response = await fetch(`${BASE_URL}/api/recipes/${id}`, {
    method: "DELETE",
    headers: authHeader(username, password),
  });
  if (!response.ok) throw new Error("Delete failed");
}

// ─── Auth ────────────────────────────────────────────────────────────────────

export async function register(username, email, password) {
  const response = await fetch(`${BASE_URL}/auth/register`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ username, email, password }),
  });
    const data = await response.json();
    if (!response.ok) throw new Error(data.error || "Registration failed");
    return data;
}

export async function login(username, password) {
  const response = await fetch(`${BASE_URL}/auth/login`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ username, password }),
  });
  const data = await response.json();
  if (!response.ok) throw new Error(data.error || "Login failed");
  return data;
}

export async function logout(username, password) {
  await fetch(`${BASE_URL}/auth/logout`, {
    method: "POST",
    headers: authHeader(username, password),
  });
}
Why localStorage? It keeps the user logged in after a page refresh. It is good enough for a classroom app. In production you would use HTTP-only cookies or a session token instead.

Step 2: minimal home page — app/page.js

The home page only does two things: fetch the recipe list and show it. It does not handle auth, forms, or routing logic — those belong on their own pages. Notice how we return different HTML based on the state - if the page is loading, or there is an error or there is actual response.

app/page.js
"use client";

import { useEffect, useState } from "react";
import { fetchRecipes } from "@/lib/api";  //<--- import the API helper function
import RecipeCardList from "@/components/RecipeCardList";  //<--- import a component to render the list
import Alert from "@mui/material/Alert"; //<--- import MUI Alert for error messages

export default function HomePage() {
  const [recipes, setRecipes] = useState([]);
  const [error, setError] = useState("");
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetchRecipes()
      .then(setRecipes)
      .catch((err) => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  return (
    <main>
      <h1>RecipeShare</h1>
      {loading ? (
        <p>Loading recipes...</p>
      ) : error ? (
        <Alert severity="error">{error}</Alert>
      ) : (
        <RecipeList recipes={recipes} />
      )}
    </main>
  );
}
Checkpoint: the page fetches and renders the recipe list. No auth, no forms — just recipes.

Step 3: add a NavBar with login / logout links

We add a NavBar that reads auth from localStorage and either shows Login / Register links for guests, or New Recipe + Logout for logged-in users. The navbar lives in layout.js so it appears on every page automatically.

components/NavBar.jsx
"use client";

import { useEffect, useState } from "react";
import { useRouter } from "next/navigation";
import { getAuth, clearAuth, logout } from "@/lib/api";
import Link from "next/link";

export default function NavBar() {
  const [auth, setAuth] = useState(null);
  const router = useRouter();

  useEffect(() => {
    setAuth(getAuth());
  }, []);

  async function handleLogout() {
    if (auth) {
      await logout(auth.username, auth.password).catch(() => {});
      clearAuth();
      setAuth(null);
      router.push("/");
    }
  }

  return (
    <nav style={{ padding: "12px 24px", background: "#0f172a", color: "white",
                  display: "flex", gap: "16px", alignItems: "center" }}>
      <Link href="/" style={{ color: "white", fontWeight: "bold" }}>RecipeShare</Link>

      {auth ? (
        <>
          <span>Hello, {auth.username}</span>
          <Link href="/recipes/create" style={{ color: "#93c5fd" }}>New Recipe</Link>
          <button onClick={handleLogout} style={{ color: "white", background: "transparent",
                  border: "1px solid white", borderRadius: "6px", padding: "4px 12px",
                  cursor: "pointer" }}>Logout</button>
        </>
      ) : (
        <>
          <Link href="/login"    style={{ color: "#93c5fd" }}>Login</Link>
          <Link href="/register" style={{ color: "#93c5fd" }}>Register</Link>
        </>
      )}
    </nav>
  );
}

Step 4: register page — app/register/page.js

The register page is a client component that holds a small form. On success it calls router.push("/login") to send the user straight to login. All API work is imported from lib/api.js — the page itself has no fetch(). We create a register directory under app/ to hold the page.js for the /register route, following Next.js conventions.

app/register/page.js
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";
import { register } from "@/lib/api";
import Alert from "@mui/material/Alert";
import Box from "@mui/material/Box";
import Button from "@mui/material/Button";
import TextField from "@mui/material/TextField";
import Typography from "@mui/material/Typography";
import Link from "@mui/material/Link";

export default function RegisterPage() {
  const [username, setUsername] = useState("");
  const [email, setEmail]       = useState("");
  const [password, setPassword] = useState("");
  const [error, setError]       = useState("");
  const router = useRouter();

  async function handleSubmit(e) {
    e.preventDefault();
    setError("");
    try {
      await register(username, email, password);
      router.push("/login");          // ← navigate to login after success
    } catch (err) {
      setError(err.message);
    }
  }

  return (
    <main>
      <Box sx={{ maxWidth: 400, mx: "auto", mt: 10, px: 3, py: 4,
                 borderRadius: 3, boxShadow: 3, backgroundColor: "white" }}>

        <Typography variant="h5" fontWeight="bold" mb={2}>
          Create account
        </Typography>

        {error && <Alert severity="error" sx={{ mb: 2 }}>{error}</Alert>}

        <Box component="form" onSubmit={handleSubmit}
             sx={{ display: "flex", flexDirection: "column", mt:2,  gap: 2 }}>

          <TextField label="Username" value={username} required fullWidth
                     onChange={(e) => setUsername(e.target.value)} />

          <TextField label="Email" type="email" value={email} required fullWidth
                     onChange={(e) => setEmail(e.target.value)} />

          <TextField label="Password" type="password" value={password} required fullWidth
                     onChange={(e) => setPassword(e.target.value)} />

          <Button type="submit" variant="contained" size="large" fullWidth>
            Register
          </Button>

          <Typography variant="body2" sx={{ textAlign: "center" }}>
            Already have an account?{" "}
            <Link href="/login" underline="hover">Login</Link>
          </Typography>

        </Box>
      </Box>
    </main>
  );
}
Checkpoint: fill the form, submit, and you should land on /login.

Step 5: login page — app/login/page.js

Login calls the login() helper, stores credentials with saveAuth(), then pushes the user back to the home page.

Basic Auth reminder: the server expects an Authorization: Basic <base64> header. The authHeader() helper inside lib/api.js builds that automatically — the page just passes username and password.
app/login/page.js
"use client";

import { useState } from "react";
import { useRouter } from "next/navigation";
import { login, saveAuth } from "@/lib/api";
import Alert from "@mui/material/Alert";
import Box from "@mui/material/Box";
import Button from "@mui/material/Button";
import TextField from "@mui/material/TextField";
import Typography from "@mui/material/Typography";
import Link from "@mui/material/Link";

export default function LoginPage() {
  const [username, setUsername] = useState("");
  const [password, setPassword] = useState("");
  const [error, setError]       = useState("");
  const router = useRouter();

  async function handleSubmit(e) {
    e.preventDefault();
    setError("");
    try {
      await login(username, password);
      saveAuth(username, password);
      window.location.href = "/";   // ← full reload so NavBar re-reads localStorage
    } catch (err) {
      setError(err.message);
    }
  }

  return (
    <main>
      <Box sx={{ maxWidth: 400, mx: "auto", mt: 10, px: 3, py: 4,
                 borderRadius: 3, boxShadow: 3, backgroundColor: "white" }}>

        <Typography variant="h5" fontWeight="bold" mb={2}>
          Login
        </Typography>

        {error && <Alert severity="error" sx={{ mb: 2 }}>{error}</Alert>}

        <Box component="form" onSubmit={handleSubmit}
             sx={{ display: "flex", flexDirection: "column", gap: 2 }}>

          <TextField label="Username" value={username} required fullWidth
                     onChange={(e) => setUsername(e.target.value)} />

          <TextField label="Password" type="password" value={password} required fullWidth
                     onChange={(e) => setPassword(e.target.value)} />

          <Button type="submit" variant="contained" size="large" fullWidth>
            Login
          </Button>

          <Typography variant="body2" sx={{ textAlign: "center" }}>
            No account?{" "}
            <Link href="/register" underline="hover">Register</Link>
          </Typography>

        </Box>
      </Box>
    </main>
  );
}

Concept: protecting a route with router.push

A protected route is a page only logged-in users should see. In Next.js we guard it at the top of the component: if there is no auth in localStorage, we call router.push("/login") immediately and render nothing. We will put this pattern to use in Lesson 3 when we build the create and edit recipe pages. 

// Pattern used inside any protected page
useEffect(() => {
  const auth = getAuth();
  if (!auth) {
    router.push("/login");    // ← redirect unauthenticated users
  }
}, []);
User visits /recipes/create → getAuth() === null? yes → router.push("/login") no → Show the form
As always, the real protection is on the server. The frontend guard is just for good UX — it prevents the user from even seeing the form when not logged in.
